Chrome Delicious Extension - Bookmark Sync

URGENT: Chrome 19 broke the login. Extension version 1.0.1.0 fixes this problem.

Do you want your Delicious bookmarks replicated to your Google Chrome Bookmark Bar? You're in the right place.
There are plenty of other extensions that do a great job of allowing you to tag pages and edit tags. We might add these features in the future... but for now this extension is all about the Google Chrome Bookmark Bar.

This extension does a one-way (two-way optional), on-demand copy from your Delicious bookmarks to the Google Chrome Bookmark Bar. This is done on a tag by tag basis - you select which tags to copy. Let's take a peek:

Depending on the bookmark folder options selected, bookmarks are sync'd from Delicious tag to Google Chrome bookmark bar folder:

Bookmark bar option: Single folder on bookmark bar option:

"Delete bookmarks that have been deleted from Delicious." - when this option is selected, bookmarks that have been deleted from Delicious will be deleted from Google Chrome during the next sync.

"Sync bookmarks created in Chrome back to Delicious" - when this option is selected, bookmarks created in Google Chrome (but only in the Delicious folders) will be created back in Delicious during the next sync. They are created as either public or private as indicated by the option selection.

"Auto Sync": select an auto sync period and the extension will sync on a regular basis. This feature requires a saved username and password.

Errors and Warnings

"Error: could not create bookmark (or other action). This request exceeds available quota."

This message is from Google Chrome. Google Chrome places some limits on how many bookmarks actions (create, edit, move, delete) an extension can do within certain time periods. One should not get this during normal usage of the extension... but it is possible when doing a lot of deleting and re-creating of bookmarks.

"Error from Delicious: 999 Unable to process request at this time - error 999"

This message is from the Delicious server. Delicious places some limits on how many service calls a user can make to the Delicious bookmark service within certain limits. Again here, one should not get this during normal usage of the extension.

Note on saving passwords: If you choose to save your password, you are going to see a warning about the risks involved with saving your password - even though it is stored in an encrypted format. That same warning can be viewed anytime by clicking the 'info' link there. Save your password at your own risk. Definitely do not save your password on a public computer.
